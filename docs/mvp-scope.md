# Jonard Cloud MVP 범위 문서

## 1. 목적

이 문서는 Jonard Cloud 1차 개발 범위를 고정하기 위한 기준 문서다. 기존 시스템의 모든 기능을 한 번에 옮기지 않고, 운영 콘솔로서 가장 먼저 사용할 수 있는 최소 기능만 MVP에 포함한다.

MVP의 목표는 다음과 같다.

- 로그인 후 권한에 맞는 콘솔을 볼 수 있다.
- 장비 목록과 상세 정보를 조회할 수 있다.
- 장비 그룹, 사용자, 연락처, 작업, 모니터링 기록의 기본 흐름을 확인할 수 있다.
- 관리자와 일반 사용자의 메뉴/권한 차이를 검증할 수 있다.
- 고급 장비 제어, 지도 편집, OTDR 파일 뷰어, 리포트 생성은 후순위로 분리한다.

## 2. MVP 원칙

- 기존 시스템의 업무 흐름은 유지한다.
- 화면 구성과 명칭은 Jonard Cloud에 맞게 새로 설계한다.
- 백엔드는 FastAPI 기준으로 API 경계를 먼저 만든다.
- 프론트엔드는 React TypeScript + Ingradient UI 기준으로 구현한다.
- DB 구조는 처음부터 확정하지 않고, 핵심 엔티티를 기준으로 점진적으로 구체화한다.
- 장비 실시간 제어는 MVP에서 실제 통신 대신 어댑터 인터페이스만 둔다.

## 3. 포함 범위

### 3.1 인증과 세션

포함 기능:

- 로그인
- 로그아웃
- 현재 사용자 정보 조회
- 권한별 메뉴 노출
- 인증이 필요한 페이지 접근 제어

MVP 기준:

- 회원가입은 만들지 않는다.
- 비밀번호 찾기, 이메일 인증, IMEI/Serial 검증은 후순위다.
- 초기에는 개발용 계정 또는 seed 계정으로 시작할 수 있다.

### 3.2 공통 콘솔 레이아웃

포함 기능:

- 좌측 또는 상단 내비게이션
- 페이지 헤더
- 검색/필터/액션 버튼을 담는 툴바
- 목록/상세/모달/토스트/확인창 패턴
- 권한별 메뉴 구성

MVP 기준:

- 기존 시스템처럼 모든 기능을 메뉴에 노출하지 않는다.
- MVP에서 구현된 기능만 메뉴에 표시한다.

### 3.3 Home

포함 기능:

- 내 장비 요약
- 최근 작업 요약
- 최근 모니터링 기록 요약
- 관리자일 경우 전체 사용자/장비 요약

MVP 기준:

- 숫자 기반의 간단한 요약 카드와 최근 목록만 제공한다.
- 차트는 간단한 placeholder 수준으로 둔다.

### 3.4 Devices

포함 기능:

- 장비 목록 조회
- 장비 상세 조회
- 장비 상태 표시
- 장비 그룹 표시
- 장비 검색
- 장비 모델/Serial/IMEI/소유자/상태 필터
- 장비 상세에서 관련 기록, 작업, 파일로 이동

MVP 기준:

- 장비 등록/수정은 관리자 중심으로 최소 구현한다.
- 실제 장비 명령 전송은 하지 않는다.
- 상세 화면에서 고급 설정값은 읽기 전용 또는 비활성 상태로 둔다.

### 3.5 People

포함 기능:

- 연락처 목록
- 사용자/연락처 상세
- 팀 또는 그룹 표시
- 장비 공유 대상 확인

MVP 기준:

- 초대 메일 발송은 실제 발송 대신 API 구조만 만든다.
- 복잡한 권한 위임은 후순위다.

### 3.6 Work Orders

포함 기능:

- 작업 목록
- 작업 상세
- 작업 상태 표시
- 장비/담당자 연결
- 작업 생성 기본 폼

MVP 기준:

- 작업 상태는 `draft`, `assigned`, `in_progress`, `completed`, `cancelled` 정도로 단순화한다.
- 기존 시스템의 Job Order와 To-do를 하나의 Work Orders 도메인으로 합친다.

### 3.7 Live Monitor / Records

포함 기능:

- 장비별 최근 record 목록
- record 상세
- 상태/시간/장비 기준 필터
- 장비 상세에서 record 목록으로 이동

MVP 기준:

- 실시간 WebSocket push는 후순위다.
- 초기에는 API polling 또는 수동 새로고침으로 처리한다.
- splice record, device event, operation log를 구분할 수 있는 구조를 둔다.

### 3.8 Files

포함 기능:

- 파일 목록
- 파일 메타데이터 조회
- 장비/작업/기록과 파일 연결
- 파일 다운로드 링크 또는 placeholder

MVP 기준:

- SOR/SOLA/GDM 전용 뷰어는 만들지 않는다.
- 파일 업로드는 후순위다.

### 3.9 Admin Tools

포함 기능:

- 사용자 목록
- 사용자 상세
- 장비 목록
- 고객/조직 목록
- 권한별 접근 제한

MVP 기준:

- 관리자 화면은 운영 확인용 CRUD 골격까지만 만든다.
- 공지, FAQ, Help CMS, 배너 등 콘텐츠 관리는 후순위다.

## 4. 제외 범위

MVP에서 제외하는 기능:

- 회원가입 전체 플로우
- Serial Number / IMEI Number 실제 검증
- 이메일 인증
- 비밀번호 찾기
- 지도 기반 Fiber/Network Map 편집
- OTDR/SOR/SOLA/GDM 전용 파일 뷰어
- PDF 리포트 생성
- Excel import/export
- MQTT/AMQP 실제 장비 제어
- 장비 펌웨어 업데이트
- 고급 장비 파라미터 설정
- 실시간 WebSocket 대시보드
- 복잡한 통계 차트
- 다국어 전체 적용
- 고객사별 커스텀 테마

## 5. 권한별 MVP 범위

### 5.1 일반 사용자

사용 가능:

- Home
- Devices
- People
- Work Orders
- Live Monitor / Records
- Files
- Account

제한:

- 전체 사용자 관리 불가
- 전체 고객/지역 관리 불가
- 다른 사용자의 비공유 장비 접근 불가

### 5.2 관리자

사용 가능:

- Home
- Devices
- People
- Work Orders
- Live Monitor / Records
- Files
- Admin Tools
- Audit Log placeholder

제한:

- MVP에서는 시스템 설정 일부만 제공
- 실제 장비 명령은 비활성 또는 mock 처리

### 5.3 지사 관리자

사용 가능:

- Home
- Devices
- Customers
- Records
- Audit Log placeholder

제한:

- 할당된 지역/고객 범위 밖 데이터 접근 불가
- 시스템 전체 설정 불가

## 6. MVP 완료 기준

MVP는 다음 조건을 만족하면 완료로 본다.

- 프론트엔드에서 인증 후 기본 콘솔에 진입할 수 있다.
- 권한별 메뉴가 다르게 보인다.
- Devices 목록과 상세가 동작한다.
- Work Orders 목록과 상세가 동작한다.
- Records 목록과 상세가 동작한다.
- Admin 사용자/장비 목록을 조회할 수 있다.
- API 응답 형식과 에러 형식이 일관된다.
- README 기준으로 프론트와 백엔드를 실행할 수 있다.
- 실제 장비 통신이 없어도 mock 데이터로 주요 흐름을 검증할 수 있다.

## 7. 우선순위

1순위:

- 인증
- 공통 레이아웃
- 권한별 메뉴
- Devices 목록/상세
- 기본 API 구조

2순위:

- People
- Work Orders
- Records
- Files 목록

3순위:

- Admin Tools
- Analytics 요약
- Audit Log placeholder

4순위:

- 지도
- 파일 뷰어
- 장비 제어
- 리포트
- Excel
